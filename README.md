# HackerRank Documentation

AI-first documentation site — Notion sources → Git → Mintlify.

## Architecture

```
Notion (Screen + Interviews page trees)
        │  (every 30 min via GitHub Actions)
        ▼
sync/sync.py  →  writes docs/*.mdx, images/, updates docs.json
        │
        ▼
Git (this repo)  →  Mintlify Pro (auto-deploys, indexes for AI chatbot)
```

- Landing page (`index.mdx`) is set to `mode: "custom"` and auto-opens the AI
  chatbot on load. Users interact primarily through chat; citations link to the
  full articles.
- Article pages use `mode: "wide"` — minimal chrome, no right-side TOC.

## Repo layout

- `docs/screen/`, `docs/interviews/` — MDX files, one per Notion page.
- `images/` — content-addressable image cache (SHA1-named files).
- `.mintlify/Assistant.md` — chatbot persona/style rules.
- `sync/` — Python sync worker. See `sync/README.md`.
- `.github/workflows/sync-notion.yml` — 30-min cron that runs the sync.
- `docs.json` — Mintlify site config (navigation is regenerated by the sync).
- `.sync-state.json` — per-page `last_edited_time` tracker for incremental sync.

## Local development

```bash
# 1. Install Mintlify CLI
npm i -g mint

# 2. Preview locally
mint dev

# 3. Run a one-off sync from Notion
python3 -m venv .venv
.venv/bin/pip install -r sync/requirements.txt
NOTION_API_KEY=<token> .venv/bin/python -m sync.sync
```

## Deploying

Push to `main` — Mintlify auto-deploys. The AI chatbot re-indexes on each
deploy.

## Editorial workflow

1. Draft articles in a *separate* Notion parent page (not under `Screen` or
   `Interviews`).
2. When the article is ready to publish, drag it into the appropriate section
   under `Screen` or `Interviews`.
3. Within 30 minutes, the sync worker picks it up and commits the MDX.
4. Mintlify redeploys automatically.

To un-publish, move the page out of `Screen` / `Interviews` — the next sync
deletes the MDX file.
