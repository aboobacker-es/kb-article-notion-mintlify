You can view the Test report of a candidate after the candidate has taken a test. For Front-end, Back-end, and Full-stack project-type questions, the report displays a Candidate's performance score based on the scoring mechanism specified. You can conduct further review and evaluation by downloading or launching the Candidate's project directly in an IDE.

**Prerequisites**

1. You must be logged in to your Hackerrank for Work account.
1. The candidate must have attempted the test, and a report must have been generated.

**Steps**

1. Click the **Tests** tab and click the required test name from the list.
1. Click on the **Candidates** option below the test name, and then click the required button from the left-hand pane that shows various **candidate status** options.

  <Frame>
    <img src="/images/front-end-back-end-and-full-stack-test-reports/69b3f66d83ca791e.png" alt="image" />
  </Frame>
1. Click on the** candidate's name **to view the report** **for the required candidate.

  <Frame>
    <img src="/images/front-end-back-end-and-full-stack-test-reports/3dc8c320cf13a2e8.png" alt="image" />
  </Frame>
1. You can view the **Summary** and **Timeline** tabs. Click each tab to view different details.
1. Upon clicking the **View Report,** you can see the test case-wise split-up of scores and download or review the Candidate's submitted code in an IDE. The report provides two options for evaluating the candidate's solution in the submitted project files.

  <Frame>
    <img src="/images/front-end-back-end-and-full-stack-test-reports/c2380fba55df87ab.bin" alt="image" />
  </Frame>
1. Select the IDE view to automatically open the Candidate's project in an IDE and review the submitted solution. You can directly change the candidate's code and run and test it to determine where the candidate made a mistake. This approach does not require downloading the project locally.

<Note>
**Note**: The IDE session is active only for 30 minutes, and any changes made during the session are not saved.
</Note>

1. A new browser window displays the updated project files, commits, and comparison of the files. You can download the project files and test the solution locally.
